# 🧩 Cómo Crear una Issue Correctamente

En este proyecto, los estudiantes también pueden proponer mejoras.

Pero una Issue mal escrita genera caos.
Una Issue bien escrita genera colaboración profesional.

---

# 🎯 ¿Qué es una Issue?

Una Issue es una tarea clara y específica que alguien puede resolver.

No es:
❌ "Mejorar Python"
❌ "Arreglar cosas"
❌ "Agregar más ejemplos"

Debe ser concreta y accionable.

---

# 📌 Reglas Obligatorias

✅ Una Issue = Un archivo o una mejora específica  
✅ Debe explicar el objetivo  
✅ Debe explicar qué se espera como resultado  
✅ Debe indicar el archivo afectado  
✅ Debe tener título claro  
❌ No crear Issues duplicadas  
❌ No crear Issues generales o ambiguas  

---

# 🏷 Formato del Título

Usar prefijos:

[DOC] → Documentación  
[FEATURE] → Nueva funcionalidad  
[REFACTOR] → Mejora de código o estructura  
[FIX] → Corrección  

Ejemplos correctos:

[DOC] Mejorar explicacion en listas.md  
[FEATURE] Agregar ejemplos de while en loops.md  
[FIX] Corregir error de sintaxis en funciones-basicas.md  

---

# 📝 Plantilla Obligatoria para Crear una Issue

Copiar y pegar este formato:

---

## 🎯 Objetivo

Explicar claramente qué se quiere hacer.

---

## 📂 Archivo afectado

Ruta exacta del archivo.

Ejemplo:
03-estructuras-de-datos/listas.md

---

## 📌 Tareas

- Tarea 1
- Tarea 2
- Tarea 3

---

## 🧠 Resultado esperado

Explicar cómo debería quedar después de resolver la Issue.

---

## 🔀 Rama sugerida

doc-tema-tuNombre

---

# 🧩 Ejemplo de Issue Bien Hecha

Título:
[DOC] Agregar ejemplos prácticos en diccionarios.md

Descripción:

## 🎯 Objetivo
Agregar ejemplos prácticos de uso real de diccionarios.

## 📂 Archivo afectado
03-estructuras-de-datos/diccionarios.md

## 📌 Tareas
- Agregar ejemplo con acceso a keys
- Agregar ejemplo con modificación de valores
- Agregar ejemplo con recorrido usando for

## 🧠 Resultado esperado
El archivo debe tener al menos 3 ejemplos nuevos explicados paso a paso.

## 🔀 Rama sugerida
doc-diccionarios-tuNombre

---

# 🚫 Ejemplo de Issue Mal Hecha

Título:
Mejorar Python

Descripción:
Agregar más cosas.

❌ No explica nada
❌ No dice dónde
❌ No es accionable
